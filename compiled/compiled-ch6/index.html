    小組共識會議辦理與紀錄流程¶

    一、正文說明¶

    小組共識會議是本中心用來統整行政布達、校準個案處遇標準、確認服務指標執行情形，以及累積團隊共識的重要固定會議。此會議的功能不只是交換資訊，而是將中央與主管機關的新規定、中心內部實務經驗、個案紀錄撰寫原則、個案管理系統操作重點及各項工作指標，透過定期會議轉化為一致做法，避免同仁在服務現場出現標準不一、理解落差或執行方式分散的情形。

    依目前中心規劃，小組共識會議原則上固定每月辦理一次，時間多安排於每月第二週星期三上午 9:30 至 12:00，或下午 13:30 至 16:00。除個案與行政事項外，會議亦承擔「建構工作手冊」與「滾動修正服務模式」的功能，因此會議記錄與列管追蹤必須完整，不能只停留在口頭討論。會議內容應留下可追蹤、可查核、可回溯的紀錄，以利後續檢視與實際執行。

    二、步驟／流程¶

    1. 會前準備¶

    1. 依年度行事曆或既定排程，先確認當月會議日期、時段與地點。
    2. 會議記錄或當月小秘書應提前整理上次會議未結案之列管事項，確認哪些事項需持續追蹤。
    3. 提前向行政確認是否有主管機關新布達、需轉知事項、表單更新或指標提醒。
    4. 蒐集當月需討論之重點，例如：
    5. 個案分級與服務頻率標準
    6. ISP 與個案紀錄撰寫問題
    7. 個管系統使用或填報疑義
    8. 各項服務目標與指標進度
    9. 跨單位合作或執行卡關事項
    10. 若需投影或共編檔案，應先準備會議檔案與列管表，避免會議中臨時整理。

    2. 會議進行¶

    1. 由會議主持人或督導說明本次會議目的與主要議程。
    2. 由小秘書或會議記錄先報告前次列管事項之追蹤情形。
    3. 依序進行行政布達、個案標準討論、系統操作共識、服務指標檢視及其他臨時提案。
    4. 討論過程中，需同步記下：
    5. 討論主題
    6. 結論
    7. 後續執行方式
    8. 主責人
    9. 預計完成時間
    10. 若某議題需後續再補資料、再提督導討論或另開會處理，應明確標示為列管事項。

    3. 會後整理¶

    1. 會後應儘速完成會議紀錄，避免遺漏細節。
    2. 紀錄內容至少應包含：會議日期、出席人員、討論事項、結論、列管事項、主責人及完成期限。
    3. 若有需留存之附件，例如會議簡報、布達內容、流程圖、指標表單，也應一併整理。
    4. 完成後送督導確認或陳核，確認版再存入正式資料夾。
    5. 次月會前，由小秘書或記錄人員再將未結事項續列追蹤。

    三、重要知識或判斷原則¶

    1. 共識會議的核心不是「開會」，而是形成一致做法¶

    會議最重要的功能是讓中心內部對分級、紀錄、系統、行政與指標有一致理解。若只有討論、沒有形成可執行結論，會議就無法達到實質效益。

    2. 會議紀錄必須能支撐後續執行¶

    小組共識會議不是單純的聊天紀錄，紀錄內容應能讓未出席者看完後理解：本次做了什麼決定、為什麼這樣做、接下來誰要做、何時要完成。

    3. 列管事項是共識延續的關鍵¶

    若會議決議沒有列管，往往會停留在當下共識，後續容易遺忘或不了了之。因此會議中凡涉及制度修正、資料補件、工具調整、行政追蹤或跨人員合作事項，都應列入追蹤。

    4. 共識會議同時具有訓練功能¶

    對新進同仁而言，共識會議也是理解中心標準、熟悉判斷邏輯與吸收實務經驗的重要場域。因此不宜只看成行政例會，而應視為團隊內部的制度校準與知識傳承機制。

    四、注意事項¶

    • 會議前應先整理上次列管事項，避免每次重頭開始。
    • 行政布達事項若涉及流程修正，應同步記錄更新點，避免口頭告知後無文件可查。
    • 討論結論若尚未成熟，應標示為待確認，不可誤當成正式規範直接執行。
    • 會議記錄完成後，應使用中心既定格式並存放至正式資料夾，不宜只留在個人電腦或私人筆記。
    • 若會議內容涉及個案或跨單位敏感資訊，記錄時應留意個資與保密原則。

    共識會議小秘書輪值與列管追蹤機制¶

    一、正文說明¶

    小組共識會議要能真正發揮追蹤與統整功能，除了主持人與與會人員外，小秘書的角色非常重要。小秘書不是單純負責打字，而是負責把會議中的討論事項、未結案問題、行政進度與後續待辦整理成可以持續追蹤的工作結構，讓會議不會每次都重頭開始。

    目前中心已明確採輪值制辦理共識會議小秘書工作，其基本精神在於分攤行政負擔、維持資訊透明，並讓每位同仁都有機會熟悉會議追蹤機制。小秘書工作的重點在於：會前確認資料、會中帶列管事項、會後完成紀錄與追蹤。因此，小秘書必須熟悉列管表使用方式、知道如何承接前次未完成事項，也要清楚哪些議題需要轉為正式追蹤項目。

    二、步驟／流程¶

    1. 會前作業¶

    1. 建立新的會議工作表或複製既有會議檔案版本。
    2. 將上次尚未完成的列管事項，複製到新表或新月份頁籤，作為本次追蹤基礎。
    3. 提醒並確認各組同仁已填寫行政進度、待報告事項或需回覆之列管內容。
    4. 先檢視是否有未補資料、未完成回覆、未送件事項，避免開會時才臨時查找。
    5. 如需投影會議表單，應先測試檔案是否可正常開啟與顯示。

    2. 會中作業¶

    1. 於會議開始後，先協助主持人帶領報告前次追蹤列管事項。
    2. 依序記錄每項列管內容目前進度，包括：
    3. 已完成
    4. 持續追蹤中
    5. 待補資料
    6. 需轉請其他人處理
    7. 會議進行中，即時記錄新增的決議與待辦事項。
    8. 凡涉及主責人、完成期限或需另行追蹤之事項，應立即補到列管表中。
    9. 若討論中有明確共識，例如流程調整、文件更新、系統欄位寫法、行政分工修正等，也應記載於紀錄內，不可只留在口頭。

    3. 會後作業¶

    1. 將會議紀錄整理完成，確認內容與會議實際結論一致。
    2. 依中心規定最遲隔天繳交督導陳核。
    3. 將確認後的會議紀錄存入正式資料夾。
    4. 若有需立即追蹤之事項，應主動提醒相關主責人員，避免只有存檔沒有執行。
    5. 下次輪值小秘書交接時，應讓接手者能看懂目前列管狀態。

    三、重要知識或判斷原則¶

    1. 小秘書的工作核心是「讓決議可被延續」¶

    若會議決議沒有被整理成列管項目，下次開會時就容易遺漏。因此小秘書實際上扮演的是會議延續與知識承接的角色。

    2. 列管事項要可追蹤，不是只寫一句話¶

    列管內容至少要包含：問題是什麼、誰負責、何時完成、目前進度如何。若只有簡短標題，後續往往無法判斷是否完成。

    3. 會議中的臨時共識也要留下痕跡¶

    很多實務做法不是來自正式公文，而是由共識會議逐步校準形成。因此會中若出現重要判斷原則或流程修正，也應留在紀錄中，後續才有可能轉為工作手冊內容。

    4. 輪值制不代表標準可以不一致¶

    即使由不同人輪值小秘書，也應維持相同的基本做法與欄位邏輯，避免每次會議紀錄格式差異過大，影響查找與承接。

    四、注意事項¶

    • 會前一定要先把未結案列管事項帶入新表，不可漏列。
    • 會議中應即時更新，不建議只靠事後回想補寫。
    • 若某議題尚未形成定論，應標示「待確認」或「續提討論」，不要直接寫成定案。
    • 會議紀錄完成後應送督導確認，不可直接當成最終正式版流通。
    • 小秘書若發現行政進度欄位未填、資料缺漏或主責不明，應先提醒再開會，減少會議中反覆確認的時間。

    團體督導會議運作與提報規則¶

    一、正文說明¶

    團體督導會議是中心用來處理困難個案、深化專業討論與支持第一線工作者的重要機制。與小組共識會議不同，團體督導的主軸不在行政布達或流程校準，而在個案處遇、專業判斷、倫理思考與服務策略的深化討論。透過固定且具結構的督導會議，中心可以讓同仁在高風險、高複雜度案件中獲得支持，避免主責人員單獨承擔判斷壓力。

    依目前中心規劃，團體督導原則上每月辦理一次，安排在每月第四週，並依固定的時段與議程進行。年度規劃中，亦要求每位同仁提報個案討論與學習分享，讓團體督導不只是「被動接受督導」，也成為共同學習、交流與累積專業經驗的場域。

    二、步驟／流程¶

    1. 會前準備¶

    1. 依年度行事曆確認當月團督時間與地點。
    2. 確認本次提報的個案討論者與學習分享者。
    3. 提報個案者應先整理案件背景、服務歷程、目前困難、已採取措施及希望討論的重點。
    4. 如涉及高風險個案、複雜家庭動力、跨網絡合作或倫理兩難，應先將脈絡整理清楚，避免會中無法聚焦。
    5. 若需使用簡報或書面資料，應事先準備並確認個資遮蔽。

    2. 會議進行¶

    1. 由督導或主持人說明本次會議安排。
    2. 依排定議程進行個案討論、學習分享、督導主題及綜合討論。
    3. 個案討論時，提報人應簡要說明：
    4. 案件基本背景
    5. 服務歷程與目前分級
    6. 主要困難與風險點
    7. 已連結或尚未連結的資源
    8. 目前猶豫或需要督導意見之處
    9. 督導與團隊成員可就服務方向、資源連結、關係建立、家庭支持、紀錄寫法與風險判斷等提出建議。
    10. 會議記錄應整理討論重點與後續建議，但不宜流於逐字式記錄。

    3. 會後追蹤¶

    1. 提報人應依會議建議回到案件中實際調整處遇方向。
    2. 若會議中形成需跨網絡協助或需再提督導確認之事項，應列入後續追蹤。
    3. 會議記錄完成後應存入正式資料夾，供後續檢閱與學習。
    4. 同仁應留意年度提報次數，避免集中於年底補提。

    三、重要知識或判斷原則¶

    1. 團督不是行政會議¶

    團體督導的核心目的在於提升專業判斷與處遇品質，不宜被行政布達或瑣碎通知佔滿。行政事項若有需要，應與共識會議分流。

    2. 個案提報重點在「可討論的困難」¶

    提報個案時，不只是重述案情，而是要明確說出目前卡住的地方，例如：服務關係難建立、家庭支持不足、資源無法銜接、紀錄寫法不確定、風險判斷有疑慮等。

    3. 團督是支持機制，不是檢討大會¶

    團體督導應讓同仁在複雜案件中獲得理解與支持，而不是形成責備壓力。討論應聚焦於如何讓服務更適切，而非單純評價個人表現。

    4. 年度提報制度有助於知識累積¶

    每位同仁固定提報個案討論與學習分享，可促進團隊在不同議題上的經驗流動，也有助於建立中心自己的實務知識庫。

    四、注意事項¶

    • 個案討論資料應注意個資保密與適度去識別化。
    • 若同一場安排個案討論與學習分享，應事先確認時間足夠，避免壓縮主要討論。
    • 不可於同一場次同時完成同一人兩種年度提報義務，仍應依中心規定分散安排。
    • 若會議中出現重要處遇方向調整，應回到案件紀錄中補充反映。
    • 個案提報人不宜只帶問題進會議，應先說明自己已嘗試過的做法，才能讓討論更具體。

    小組聚會、外聘督導與跨網絡會議規劃¶

    一、正文說明¶

    除小組共識會議與團體督導外，中心在年度運作中還包含小組聚會、外聘專業督導、教育訓練工作坊與跨網絡聯繫會議等支持性與協作性會議機制。這些安排雖不全是日常行政流程，但都與團隊穩定、專業成長及跨單位合作密切相關，因此應在工作手冊中留有清楚的基本規劃脈絡，讓新人知道這些會議或活動為何存在、何時大致辦理、各自扮演什麼功能。

    小組聚會主要著重團隊支持與凝聚；外聘督導聚焦特定專業議題的深化；教育訓練工作坊強調知能精進與紓壓並行；跨網絡聯繫會議則用來處理跨單位合作與區域身心障礙服務之銜接問題。這些機制相互補充，使中心運作不只停留在處理日常案件，而能在專業、行政與團隊支持上維持穩定。

    二、步驟／流程¶

    1. 小組聚會規劃¶

    1. 原則上每月辦理一次，時間依當月行程安排。
    2. 由同仁輪流規劃活動形式，例如下午茶、散步、輕量戶外活動或其他支持性聚會。
    3. 目的在於促進同仁關係、減緩工作壓力、增強團隊支持感。
    4. 活動不以正式訓練為主，但仍應兼顧安全與出席安排。

    2. 外聘督導與專業課程規劃¶

    1. 依年度計畫先排定外聘督導場次、主題、講師與時間。
    2. 主題可依實務需要安排，例如：自立生活、學校資源、身障服務架構、精神疾病、性侵害預防、法律知能等。
    3. 對於特定專業業務，如監護或輔助宣告、雙老家庭、高照顧負荷、教育轉銜等，可透過外聘督導補充現場較難自行內化的知能。
    4. 如需講義、簡報或課前資料，應提前向講師確認。

    3. 教育訓練工作坊規劃¶

    1. 依年度計畫辦理工作坊形式之教育訓練。
    2. 可採上午紓壓活動、下午專業課程之雙軌設計，兼顧充電與實務知能。
    3. 若為對外開放之課程，應同步處理報名、簽到、講義、場地與成果紀錄。

    4. 跨網絡會議規劃¶

    1. 依計畫安排辦理跨網絡會議，邀請三區及相關網絡單位參與。
    2. 會議目的在於檢視服務輸送、角色定位、合作機制與處遇流程。
    3. 若涉及個案研討，應事前備妥必要資料並遵守個資保密原則。
    4. 會議後應留下紀錄、決議與必要追蹤事項，以利後續合作落實。

    三、重要知識或判斷原則¶

    1. 團隊支持也是服務品質的一部分¶

    小組聚會看似與案件無直接關聯，但對降低耗竭、維持凝聚力與支持高壓工作情境非常重要，因此不宜視為可有可無的附屬活動。

    2. 外聘督導的價值在於補足內部盲點¶

    外部專家能提供不同視角與實務經驗，特別在中心新制度建立期、處遇模式調整期或遇到特殊議題時，能幫助團隊校準方向。

    3. 跨網絡會議重點在明確分工與合作機制¶

    若會議後沒有形成轉介窗口、角色分工或後續追蹤方式，就容易流於單次交流。真正重要的是讓不同系統知道彼此如何接手與合作。

    4. 會議規劃要兼顧實務與可執行性¶

    安排主題時，應優先選擇和中心當前需求高度相關的議題，而不是只追求形式上辦理完成。

    四、注意事項¶

    • 小組聚會雖較輕鬆，仍應注意出席安排與工作銜接。
    • 外聘督導與工作坊若涉及講師邀請、講義、簽到、便當與成果資料，應提前分工。
    • 跨網絡會議如需正式發文或邀請函，應配合既定行政流程提早作業。
    • 若會議有形成明確決議，應回到共識會議或工作手冊中續作整理，避免知識停留在單次會議。

    3. 於共識會議前完成資料整理
    4. 在共識會議中報告進度與列管事項
    5. 必要時提醒其他同仁補件、補填或補資料

    一、目的¶

    二、會議目的¶

    共識會議主要用於討論：

    • 中心行政規範
    • 運作模式
    • 個案服務標準
    • 資源運用
    • 各組行政業務報告
    • 小行政進度控管

    若主要討論的是個案處遇困難，原則上應改至團體督導會議進行，而非以共識會議處理。

    三、會議時間¶

    • 原則上固定於每個月第二週辦理
    • 確切的上午或下午時段，依當次安排調整
    • 常見時間帶為：
    • 09:30–12:00
    • 13:30–16:00

    四、紀錄人員採輪值制¶

    共識會議紀錄人員採輪值制。
    負責紀錄的同仁，不只是會議中打字，而是需完成會前、會中、會後三段工作。

    五、紀錄人員職責¶

    1. 會議前 3 天¶

    應於指定 Excel 檔案中建立本次會議的新工作表，並將前次尚未結案的列管事項複製過來，作為本次會議基礎。

    2. 會議前¶

    應提醒並確認各組同仁已填妥各自行政進度。
    若有資料空白、附件缺漏或列管事項未更新，應先提醒補正。

    3. 會議中¶

    應負責： - 投影 Excel 紀錄表畫面 - 報告前次列管與本次列管事項 - 即時更新討論結果 - 整理需後續追蹤之重點

    4. 會議後¶

    應整理並完成會議紀錄，提交督導簽核。
    若依現行要求，應於會後最遲隔天或三日內完成紙本陳核，仍以中心最新共識會議要求為準。

    六、各組資料填寫期限¶

    各組負責行政業務的同仁，原則上應於會議前一天的下班前完成資料填寫。
    若另有當月特殊提醒，例如要求在會議前一天中午前完成，則以當次公告為準。

    重點不是死記某一個小時點，而是： - 不能拖到會議當天才補 - 紀錄人員要有足夠時間整併 - 各組要先對自己的內容負責

    • 已完成事項

    • 本次新增列管
    • 必要之附件、數據或路徑
    • 需要督導或其他組協助之事項

    九、列管事項處理原則¶

    1. 前次列管要持續追¶

    會議不是每月重新開始，而是要把前次未結案事項接續追蹤。

    2. 本次列管要具體¶

    新增列管時，應盡量明確寫出： - 是哪一項工作 - 誰負責 - 預計何時完成 - 下一步是什麼

    3. 解除列管要有依據¶

    若某事項解除列管，應能說明： - 已完成什麼 - 為何可解除 - 是否仍有後續延伸事項

    十、常見錯誤¶

    • 紀錄人員只做會中紀錄，沒有提前整理前次列管
    • 各組拖到會議當天才補資料
    • 只寫進度，沒寫列管狀態
    • 路徑、附件或數據不完整
    • 解除列管沒有寫清楚完成理由
    • 會後沒有即時完成陳核

    共識會議表單建立、複製列管與投影操作提醒¶

    一、目的¶

    本篇用於整理共識會議紀錄輪值人員在實務上最常操作的三件事：

    1. 建立本次會議新工作表
    2. 複製前次尚未結案的列管事項
    3. 會議中投影並即時更新紀錄表

    本篇重點放在「操作提醒」，讓接手輪值的新手不只知道制度要求，也知道實際上應先做哪些動作。

    二、這篇適合誰看¶

    本篇特別適合： - 首次輪到做共識會議紀錄的人 - 臨時接手會議紀錄的人 - 想先熟悉共識會議表單操作的人 - 需協助檢查列管事項是否正確延續的人

    三、基本前提¶

    依中心現行共識會議規則，紀錄人員原則上需於：

    • 會議前 3 天：建立新工作表，並複製前次未結列管
    • 會議前：確認各組已填妥行政進度
    • 會議中：投影表單並更新列管
    • 會議後：整理完成並送督導簽核

    因此，表單操作不是會議當天才開始，而是會前就要先準備。

    共識會議主要路徑如下：

    FS 內部伺服器指定資料夾（公開版不列示完整路徑，請依內部公告或督導提供版本查找）

    四、建立新工作表的基本做法¶

    1. 先開啟最新版主檔¶

    先進入共識會議資料夾，確認自己打開的是目前實際使用的最新版，而不是舊副本或臨時下載版本。

    2. 複製前次工作表¶

    若目前紀錄方式為一場會議一個工作表，建議可先複製前次工作表，再改成這次日期或場次名稱。
    這樣可保留既有欄位格式、顏色、公式與版面。

    3. 工作表命名要清楚¶

    命名原則建議至少讓人一看就知道： - 是哪一次會議 - 是哪個月份 - 是否為正式版或暫存版

    避免只命名成「新表1」、「複製表」等難以辨識名稱。

    五、複製前次未結列管事項¶

    1. 先找出前次列管區塊¶

    複製前，不要整張表全部搬過來。
    應先看清楚哪些屬於： - 已解除事項 - 持續追蹤事項 - 本次需延續列管事項

    2. 只複製尚未結案者¶

    原則上只複製： - 尚未完成 - 持續追蹤中 - 督導要求延續列管 - 需要各組持續回報之事項

    3. 複製後要檢查欄位對齊¶

    常見問題是複製後： - 欄位跑掉 - 顏色不一致 - 公式被覆蓋 - 負責人欄位移位

    因此複製完一定要再逐欄檢查。

    4. 新舊列管要分清楚¶

    建議在版面上能清楚分辨： - 前次延續事項 - 本次新增事項

    避免會議中看到一堆列管，卻無法區分哪些是舊案、哪些是新案。

    六、會議前的檢查重點¶

    在會議開始前，紀錄人員建議先快速檢查：

    1. 各組是否已填資料
    2. 空白區塊是否過多
    3. 路徑、附件、數據是否有缺
    4. 前次列管是否已正確複製
    5. 當次新工作表名稱是否正確
    6. 投影設備是否可正常使用

    七、投影操作提醒¶

    1. 會前先打開表單¶

    不要等會議開始才開檔。
    建議會前就先： - 開好 Excel - 停在這次工作表 - 調整字體大小 - 把畫面捲到適合投影的位置

    2. 投影時以看得清楚為主¶

    投影畫面要讓整桌同仁看得清楚，因此可依實際情況調整： - Excel 縮放比例 - 欄寬 - 列高 - 凍結窗格（如有需要）

    3. 記錄與報告同步¶

    紀錄人員不只是打字，也常需要同步報告前次列管進度，因此會前自己先熟悉列管內容會比較順。

    八、會議中更新的原則¶

    1. 優先記重點，不追求逐字¶

    共識會議紀錄不是逐字稿。
    重點是記清楚： - 本次決議 - 新增列管 - 負責人 - 預計完成方向 - 需追蹤事項

    2. 有責任人就寫清楚¶

    會議中若有明確分工，應盡量記出： - 由誰處理 - 處理什麼 - 後續要回報什麼

    3. 不確定就當場確認¶

    若聽不清楚、責任人不明或列管內容模糊，建議當場簡短確認，避免會後才發現不知在寫什麼。

    九、會後整理提醒¶

    會後不只是存檔而已，還應：

    1. 再檢查本次列管是否完整
    2. 補齊會議中來不及整理的文字
    3. 確認工作表已存到正確主檔
    4. 避免只存在個人桌面或下載資料夾
    5. 依中心規定送督導簽核

    十、常見錯誤¶

    • 打開錯版本主檔
    • 把整張舊表全部複製，導致內容混亂
    • 已解除事項也一起搬過來
    • 投影前沒先調整畫面，會議中才慌張處理
    • 會後只存本機，沒存回主檔
    • 公式、格式被覆蓋但沒發現

    十一、注意事項¶

    • 共識會議表單屬共用主檔，操作前要有版本意識。
    • 只複製需要追蹤的列管，不要整份搬。
    • 會議中重點在清楚，不在華麗。
    • 若表單格式後續更新，本篇也應同步修正。
