    第一篇 到職與新人訓練¶

    報到前準備與應備資料¶

    本節說明新人於正式報到前，應先備妥哪些基本資料、這些資料分別用於哪些行政程序，以及中心現行實際的資料傳遞方式。 報到前資料若先準備完整，報到當日才不會因勞健保加保、人事建檔、薪資匯款或系統申請前置資訊不足而延誤後續流程。對新人而言，這一步的重點不是把資料「帶來就好」，而是要先知道哪些資料是必要欄位、哪些資料會被用在不同工作，以及資料完成後應交由誰先確認。 依中心現行整理資料，報到前的核心任務可分成四類：到職時間確認、勞健保加保資料、人事資料、薪資匯款資料。原則上，這些資料應在正式到職前先蒐集完成，並依既定流程轉知承辦，讓報到日能直接銜接契約簽署、系統申請與行政庶務建置。

    二、步驟／流程¶

    1. 先確認正式到職時間¶

    新人一旦確定錄取並進入報到準備階段，第一步應先確認正式到職日期與到職時間。 此步驟的用途不只是安排第一天時間，也與勞健保加保、人事建檔、座位與設備準備、報到日流程安排有關。 實務上，應先完成： - 1. 與中心確認正式到職日期。 - 2. 確認報到當日大致時段與流程安排。 - 3. 若有特殊情況，例如須延後報到、先補件、尚待辦理帳戶或畢業文件者，應先主動告知。

    2. 準備勞健保加保所需資料¶

    報到前須先備妥勞健保加保所需之基本個人資訊，以利到職後進行加保程序。 應準備資料如下： - 身分證字號 - 出生年月日 辦理方式： - 1. 先確認資訊是否正確。 - 2. 若需以電子方式提供，應用可辨識且不易誤讀的格式提供。 - 3. 若承辦要求補充其他加保資訊，應一併確認並補齊。

    3. 準備人事資料¶

    報到前另須準備中心建檔使用之基本人事資料。 應準備資料如下： - 畢業證書掃描檔 - 履歷 辦理方式： - 1. 畢業證書以清楚可辨識之掃描檔為原則。 - 2. 履歷以目前最新版本為準。 - 3. 若學歷、姓名、聯絡資訊曾有異動，應先確認是否與目前使用資料一致。

    4. 準備薪資匯款資料¶

    薪資匯款部分，依現行資料應先準備台銀帳戶封面，作為薪資匯款建檔依據。 應準備資料如下： - 台銀帳戶封面 辦理方式： - 1. 若尚未申辦臺灣銀行帳戶，應於報到前先辦理。 - 2. 提供帳戶資料時，應確認帳號、戶名清楚可辨識。 - 3. 若帳戶尚未完成，應主動告知承辦預計補件時間。

    5. 依現行做法交付資料¶

    依現行資料，報到前蒐集完成之資料，應依序進行以下處理： - 1. 確認到職時間。 - 2. 取得身分證字號、出生年月日。 - 3. 取得畢業證書掃描檔與履歷。 - 4. 取得台銀帳戶封面。 - 5. 將上述資料轉知承辦人員辦理後續建檔與報到準備。 依現行資料，資料蒐集後會轉知鳳珠姊，並將相關資料放入 FS；若當下中心另有指定承辦，則以當下承辦人員為準。

    三、重要知識或判斷原則¶

    1. 報到前資料不是同一件事的重複要求¶

    雖然新人常會感覺身分、學歷、帳戶等資料只是「基本資料」，但在實際行政流程中，這些資料分別對應不同用途： - 身分證字號、出生年月日：主要用於勞健保加保等基本人事程序。 - 畢業證書掃描檔、履歷：主要用於人事建檔與資格確認。 - 台銀帳戶封面：主要用於薪資匯款建檔。 因此，報到前若缺任一項，都可能影響不同後續流程，不能視為可到場後再慢慢補。

    2. 報到前先準備完整，能減少報到日壓力¶

    報到當天通常還要進行薪資與勞健保說明、契約簽署、系統申請表填寫、群組與工具加入、行政庶務與環境建置。若資料未先備妥，容易讓第一天流程被打斷，也增加承辦與督導往返確認的成本。

    3. 若資料未齊，應先說，不要等到報到當日才提¶

    例如尚未辦妥台銀帳戶、畢業證書尚未掃描、履歷仍在更新等，均不應到報到當天才臨時說明。 較好的做法是先告知承辦目前缺哪些資料、何時可補件，讓中心提早調整。

    4. 目前承辦與轉知做法屬中心內部版本¶

    依目前匯入資料，報到前資訊蒐集完成後，會轉知鳳珠姊，並放入 FS。此為目前已掌握之內部做法；若之後中心調整承辦窗口，實務上仍應以最新承辦安排為準。

    四、注意事項¶

    • 務必先確認到職時間，不要只記日期而忽略當日時段安排。
    • 台銀帳戶若尚未辦理，應優先處理，避免影響薪資匯款建檔。
    • 畢業證書宜以清楚掃描檔提供，不宜僅提供模糊拍照版本。
    • 履歷請提供最新版本，避免舊資料與目前實際資訊不一致。
    • 若已有資料變更，例如姓名、電話、聯絡方式、學歷補件等，應主動告知。
    • 報到前資料屬個人與人事資訊，提供與傳遞時應注意個資保護。
    • 若中心現場指定承辦與既有資料不同，應優先依當下承辦指示辦理。

    2. 三項系統屬後續工作必備系統¶

    這三項系統與中心後續個案管理、福利查詢、社政資料查核有密切關聯，因此報到日即應先提出申請。若延後處理，後續新人訓練或實務操作時，容易因無權限而卡住。

    3. 表單雖由新人填寫，但承辦流程不可省略¶

    依中心現行做法，表單會由黃梅列印，新人填寫後再交由黃梅協助申請；帳號開通後，再由督導或承辦轉知新人。 因此，新人不宜自行跳過承辦流程，也不宜自行假設表單送出後就等於可立即使用。

    4. 契約與系統申請最好在同一天完成¶

    報到日若能把契約、保密文件與主要系統申請一次完成，後續的軟體加入、群組建置、第一週訓練與實務導入會更順。 若其中一塊延誤，後面常會連動影響。

    • 契約與保密契約如有內容不清楚，應先詢問再簽。
    • 明珠督導核章前，應先自行確認當日應辦事項是否完成，避免反覆退回補件。

    新人行政庶務與工作環境建置¶

    本節說明新人於報到當日及初期，除人事與系統申請外，尚需完成哪些行政庶務與工作環境建置事項。 對新人而言，正式進入工作現場，不只是「資料已交、契約已簽」，還包括座位、辦公用品、鑰匙、職章、輪值設備與基本環境熟悉等。這些看似零碎的庶務，其實直接影響新人是否能順利進入日常工作節奏。 依中心現行資料，行政庶務大致分成兩類：一類是由新人提供或確認後，再由行政辦理的項目；另一類是與工作座位、用品與環境熟悉有關，需由新人實際完成或配合處理的項目。這些事項若於報到初期一次完成，後續加入訓練、接聽電話、輪值與處理庶務時會較順利。

    二、步驟／流程¶

    1. 先完成個人行政資料交付¶

    新人報到初期，應先完成與個人行政識別相關之項目，包含： - 繳交大頭貼（名牌製作） - 知悉制度說明及告知內容 其中，大頭貼主要用於名牌或辨識用途；制度說明則是讓新人先理解中心基本運作與行政規範。

    2. 轉由行政處理個人工作物件¶

    依現行資料，以下項目在新人確認或提供必要資料後，由行政續辦： - 刻職章 - 辦公室鑰匙 - 辦公室遙控器（輪班者） - 零用金及文具店預存金（非必要） 辦理流程如下： - 1. 先由新人確認自己是否需配發相關物件。 - 2. 需要者由行政續辦。 - 3. 完成後再確認是否已實際領取或可使用。

    3. 確認座位與工作位置¶

    依目前報到確認表內容，報到初期尚需處理： - 確認座位（如附件座位表） - 組裝書架及座位（自行組裝） - 領取辦公室用品 - 認識環境 辦理方式： - 1. 先依座位表或現場安排確認自己的座位位置。 - 2. 若座位或書架需自行組裝，應於初期完成，避免影響後續放置資料與工作物品。 - 3. 領取日常辦公用品，確認目前可基本辦公。 - 4. 熟悉接待區、電話位置、督導座位、影印區、物資位置與基本動線。

    4. 完成環境熟悉¶

    「認識環境」不只是知道自己的桌子在哪裡，而是至少要理解： - 辦公區與接待區位置 - 公務電話與值日相關位置 - 常用文具與耗材放置處 - 影印、掃描、列印設備位置 - 督導與行政窗口大致位置 - 重要資料夾、表單與設備常見放置區

    三、重要知識或判斷原則¶

    1. 行政庶務不是附帶事項，而是新人能否順利上手的基礎¶

    若新人報到後只完成簽約與填表，卻未同步處理座位、文具、鑰匙、遙控器、名牌與制度告知，實際進入工作時仍會出現很多卡點。 因此，行政庶務雖非核心業務，但在新人導入階段是必要基礎。

    2. 有些項目由行政續辦，但新人仍需主動確認¶

    例如職章、鑰匙、遙控器、預存金等，雖由行政續辦，但新人仍應知道自己是否需要、目前是否已處理、何時可取得。 若完全被動等待，後續很容易到實際需要時才發現尚未完成。

    三、重要知識或判斷原則¶

    1. 工具與系統要分「先加入」與「先申請」¶

    TimeTree、LINE 群組、部分電話工具可先加入或先安裝；社政系統則必須先填表、送件、待開通。 新人若未先分清楚，最常見的誤解就是「我知道網址了，為什麼還不能登入」。

    2. 三個社政系統不是新人自行送件就算完成¶

    依中心現行做法，申請表由黃梅列印，新人填寫後再交由黃梅協助申請；帳號開通後，再由督導或承辦轉知。 也就是說，這類系統的申請流程本質上是「新人填表＋中心內部協助送件與追蹤」，不是新人自己列印自己送件。

    3. 官方 LINE 的命名規則是管理工具，不只是格式要求¶

    官方 LINE 一旦大量使用後，若命名混亂，後台幾乎無法快速搜尋與辨識。 因此，分機、稱謂、姓名、案主名與地區的命名方式，是為了讓大量案家管理時仍能快速找到人，不應視為可有可無。

    4. 電話工具分成桌機與手機端兩條線¶

    iTalk 偏向手機端使用，IP Phone 偏向桌機與辦公室現場使用。 新人初期至少要先知道兩者都存在，且後續值日、接待、聯繫與轉接都有可能用到。

    5. 第一章只處理「先進入可用狀態」¶

    本篇不等於完整教學手冊。 也就是說，第一章的任務是讓新人先完成加入、安裝、申請、登入與基本理解；至於完整操作、案例使用、系統登打與錯誤排除，應再回到各工具章節與訓練教材細讀。

    新人第一週訓練與課程總覽¶

    新人到職後的第一週，不以正式派案為目標，而是以「先進入工作環境、先掌握中心運作基本脈絡、先完成工具建置與基礎理解」為核心。這一週的功能，不只是安排幾堂課讓新人聽過，而是協助新人建立一條可追溯的入門路徑：先完成報到與設備建置，再認識協會與中心定位，接著進入行政、系統、電話、訪視、紀錄、資源與分級概念，最後在第一週結束前完成派案前準備與問題收斂。 依中心現行既有規劃，第一週定位為「純學習與建置週」，原則上不安排正式派案。新人在這一週應完成的，不只是上課本身，而是建立後續可進入實務的最低條件，包括：主要帳號可登入、知道常用路徑、理解中心服務範圍與角色、知道基本行政節奏、了解個案系統與電話系統用途，並對 A／B／C 案件與 1 至 4 級服務分級有初步概念。 因此，本篇的用途不是把課表原樣貼上，而是說明第一週為何這樣安排、每堂課在整體訓練中的位置是什麼、新人至少要學到什麼，以及搭配哪些教材與檔案。新人閱讀本篇後，應能大致回答以下問題： - 1. 第一週為何不正式派案。 - 2. 每天主要在學什麼。 - 3. 哪些是制度性基礎、哪些是工具性基礎、哪些是後續接案前的必要預備。 - 4. 若之後需要回頭複習，應去哪些檔案、教材或資料夾找資料。

    二、步驟／流程／指標¶

    1. 第一週整體安排原則¶

    第一週原則上採以下順序： - 1. 先完成報到與工作環境建置。 - 2. 再認識協會、中心定位與服務範圍。 - 3. 再進入行政庶務、工具與系統基礎。 - 4. 再理解電話、訪視、紀錄、資源與分級邏輯。 - 5. 最後收斂到排班、接聽電話演練與下週派案前準備。 這樣的安排順序，是為了避免新人在尚未理解中心角色、系統路徑與行政節奏前，就被迫直接面對實務案件，造成資訊破碎、提問重複或操作錯誤。

    三、重要知識或判斷原則¶

    1. 第一週的目的不是要新人立刻獨立作業¶

    第一週真正的目標，是完成「進入工作現場前的基礎建置」。若把第一週誤解成只是課程排滿的一週，容易忽略它其實是後續所有實務操作的地基。

    2. 課程順序本身就是訓練邏輯¶

    先認識環境，再認識中心，再理解行政與工具，再進入系統、訪視、資源與分級，最後做派案前準備。這個順序不宜任意打亂，否則新人容易出現知道一些零碎操作，但無法形成整體工作理解的狀況。

    3. 每堂課都有「最低學習門檻」¶

    第一週不是要求新人把所有事情做到精熟，而是至少達到可以進入下一階段的基本理解。例如： - 會登入主要工具 - 知道表單與檔案大概去哪裡找 - 知道中心角色與服務範圍 - 知道哪些問題應該向誰提問 - 對案件與分級有初步概念

    4. 第一週學習內容之後會反覆使用¶

    第一週很多看似基礎的內容，後續都會一直重複出現，例如：TimeTree、FS、電話工具、系統申請、服務紀錄、A／B／C 與 1–4 級分級。因此這些內容不是只上過一次課就結束，而是之後會在實務中持續被提取與驗證。

    四、注意事項¶

    • 第一週原則上不安排正式派案，但不代表新人可以被動只聽課，仍應主動記錄、提問與整理。
    • 課程內容若有當年度微調，仍應以中心實際排課與督導說明為準。
    • 第一週學習過程中，若發現某項工具尚未開通、某份表單未取得或某條路徑無法開啟，應立即回報，不宜延後到正式接案後才處理。
    • 新人應在第一週即開始形成自己的學習筆記或共筆，避免到第二週進入實務後才回頭補記，造成遺漏。
    • 若第一週課程因會議、活動或其他中心事務而調整順序，應優先保留報到、工具建置、行政與分級基礎等關鍵課程。

    新人培訓機制與後續銜接¶

    新人培訓不是只有報到日與第一週課表而已，而是一套從到職前、報到日、第一週純學習、第二週起逐步接觸實務，一直到滿月與第三個月仍持續運作的導入機制。若第一章只寫到第一天或第一週，雖然能讓新人知道眼前要做什麼，卻不足以回答更重要的問題：接下來會怎麼被帶、什麼時候開始接電話、什麼時候開始派案、初期會接什麼案件、遇到問題時有哪些回饋與檢核節點。 本篇的目的，就是把「新人剛進來之後，接下來會怎麼被帶進工作」寫清楚，讓新人知道中心目前採取的是分階段、漸進式、有檢核點的訓練機制，而不是報到完就立即完全獨立。這不只是保護新人，也是在保護整體服務品質與風險控管。 依中心現行規劃，新人培訓至少包含以下幾條主軸： - 1. 四週集中訓練。 - 2. 首週純學習與建置，不正式派案。 - 3. 自 5/11 起漸進式進入電話接聽與正式派案。 - 4. 三個月共筆共構制度。 - 5. 小行政訪談機制。 - 6. 依風險、服務頻率與案量進行較公平的加權配置。 因此，本篇不只是課程延伸說明，而是新人的整體培訓路徑圖。

    二、步驟／流程／指標¶

    1. 整體培訓分階段概覽¶

    依目前規劃，新人培訓大致分為下列階段：

    階段 0｜到職前／報到日¶

    重點： - 完成人事資料、契約、系統申請、群組與設備建置 - 確認工作環境、承辦窗口與基本工具 主要產出： - 報到核對表 - 帳號建置進度 - 工作環境熟悉

    階段 1｜第 1 週（純學習與建置）¶

    重點： - 基本知識 - 行政庶務 - 主要系統與電話工具基礎 - 排班與接聽電話前準備 主要產出： - 主要帳號可登入 - 課程筆記或共筆起始內容 - 可進入下一週的基本理解

    階段 2｜第 2 週¶

    重點： - 個案管理流程 - A／B／C 案件時程 - 1 至 4 級服務分級 - 紀錄與評估基礎 - 在陪同下接觸部分流程 主要產出： - 能在陪同下進行部分操作 - 開始理解實際案件節奏

    階段 3｜第 3 週¶

    重點： - 3 級電話案件練習 - 舊案閱讀 - 電話紀錄與風險辨識 - 小行政合作模式訪談開始 主要產出： - 初步電話紀錄 - 問題清單 - 小行政訪談紀錄起始

    階段 4｜第 4 週¶

    重點： - 整合應用 - 問題修正 - 滿月檢核準備 - 後續案量與學習規劃 主要產出： - 滿月回顧 - 督導檢核或個別提醒

    階段 5｜第 2 至第 3 月¶

    重點： - 持續共筆 - 每週提問與回饋 - 小行政訪談補完 - 逐步擴增案量 - 將教育過程沉澱成手冊素材 主要產出： - 三個月共筆成果 - 小行政訪談紀錄 - 工作手冊補強素材

    2. 首週純學習與建置，不安排正式派案¶

    依既有規劃，第一週的主要原則是「先打底，不正式派案」。其原因如下： - 1. 新人尚未完成主要系統、帳號、群組與工具建置。 - 2. 尚未建立對中心角色、服務範圍與行政節奏的基本理解。 - 3. 尚未理解 A／B／C 進案邏輯與 1 至 4 級服務邏輯。 - 4. 若在此時直接承接正式案件，容易形成資訊破碎、提問混亂與風險判斷不足。 因此，第一週即使課程很多，也不應被視為「還沒開始工作」，而是已在進行工作導入，只是工作形式以學習、建置與理解為主。

    3. 自 5/11 起正式進入實務，但採漸進式上線¶

    依目前訓練設計，自 5/11 起開始正式派案與接聽電話，但並非一開始就讓新人單獨承擔高風險案件，而是採「漸進式上線」原則。

    初期原則¶
    • 不以新人單獨承擔高風險案件為前提。
    • 先從較適合作為學習入口的 3 級電話案件開始。
    • A 級評估與一級案件，優先以陪同、跟訪、共訪方式帶入。
    • 正式派案前，須有基本工具可用、系統能登入、並完成第一週主要課程基礎。
    新人初期雙軌承接方式¶
    （1）3 級電話案件¶

    用途： - 練習電話訪談 - 練習電話紀錄 - 練習舊案閱讀與案情整理 - 練習與案家建立初步專業聯繫

    （2）A 級評估案與一級服務案陪同¶

    用途： - 實際觀察高需求案件之節奏與風險 - 理解訪視現場與督導／資深社工如何判斷 - 透過跟訪與共訪建立現場感

    4. 重要實務銜接節點¶

    5/11 當週｜正式派案與電話接聽開始¶

    此週為新人正式進入實務的起點。原則上會配置較適合作為學習入口之 3 級電話案件，並安排舊案交接閱讀與資深同仁回饋。

    5/18 當週｜開放跟訪¶

    安排督導訪視或資深同仁實地跟訪，讓新人對訪視節奏、現場狀況與應對方式有更具體觀察。

    5/26｜首次個督¶

    針對 5/11 起的接案、電話接聽與初步問題，進行個別回饋與後續案量調整。這一節點的功能，是將新人早期卡點集中整理，避免問題在無人梳理下持續累積。

    滿月時點｜滿月團督與工作手冊使用回饋¶

    滿月檢核不只是看新人「會不會做」，也要看新人目前使用手冊與共筆的狀況、實務遇到的障礙，以及哪些地方仍需補寫或修正。¶

    5. 第二週至第四週的訓練重點¶

    第 2 週｜個案管理與評估分級¶

    重點： - 需求評估流程 - A／B／C 案件時程 - 開案與不開案判斷 - 1 至 4 級服務分級 - 家系圖與服務紀錄基本格式 實作方式： - 在陪同下觀摩評估 - 練習紀錄撰寫 - 進行部分電話聯繫 檢核方式： - 督導抽查紀錄 - 口頭提問 - 操作示範

    第 3 週｜電話關懷與案件接手¶

    重點： - 3 級案件電話訪談 - 電話關懷紀錄 - 官方 LINE 連結 - 舊案資訊理解 - 風險辨識與轉介邏輯 實作方式： - 由舊人移轉部分較適合作為練習的案件 - 新人開始實際進行電話訪談與開案練習 檢核方式： - 至少完成指定電話練習與紀錄 - 接受回饋與修正

    第 4 週｜整合應用與滿月檢核¶

    重點： - 小行政合作模式整理 - 跨組合作問題整理 - 工作手冊使用與回饋 - 常見錯誤修正 - 個別培訓紀錄與提醒 實作方式： - 整理共筆 - 提出問題與後續學習需求 - 準備進入滿月檢核 檢核方式： - 滿月團督 - 督導檢核表或口頭檢核

    6. 共筆制度¶

    依目前規劃，新人到職起三個月內，應持續進行共筆記錄。

    共筆目的¶
    • 把學習過程留痕
    • 將重複提問轉化成標準答案
    • 為工作手冊後續補寫提供素材
    • 協助督導理解新人卡點與進度
    共筆內容¶
    • 學習摘要
    • 行政流程
    • 系統操作步驟
    • 實務觀察
    • 小行政合作經驗
    • 待補內容
    共筆頻率¶
    • 原則上每日記錄
    • 若當日無完整記錄，也應至少留下重要問題與待追問點

    7. 小行政訪談機制¶

    新人除一般課程與實務接案外，尚須依安排與各組小行政至少進行一次格式化訪談。

    訪談目的¶
    • 釐清各小行政的目的、啟動條件與合作流程
    • 把原本口耳相傳的隱性知識整理成可追溯內容
    • 作為共筆與手冊更新的材料
    訪談重點¶
    • 此項小行政的核心功能是什麼
    • 何種情況下需要啟動
    • 啟動前需準備哪些資料
    • 要交給誰、用什麼方式傳遞
    • 常見遺漏與合作期待是什麼
    訪談後處理¶
    • 由新人整理訪談紀錄
    • 交受訪者確認
    • 送督導複核
    • 納入共筆資料庫

    8. 案件承接與案量配置原則¶

    新人案量配置並非平均分配即可，而是要考量風險、服務頻率、行政負荷與現有案量。

    目前已知原則¶
    • 第一週不安排正式派案。
    • 5/11 起正式派案，但採漸進式授權。
    • 新人初期以較適合作為教學入口之 3 級電話案件優先。
    • A 級案件與一級案件以陪同、跟訪、共訪方式導入。
    • 新人擴增案量應視四週檢核、第二月表現與共筆品質調整。
    • 不宜一次性大量移入案件。
    加權平衡概念¶

    依目前規劃，督導會盤點全員案件，依風險、服務頻率、行政負荷與目前案量進行較公平的配置。這代表新人接案，不是單純按人頭分，而是要考量案件性質與學習時機。¶

    三、重要知識或判斷原則¶

    1. 新人培訓是制度，不是臨場救火¶

    若把新人培訓理解成有空才教、遇到問題再補，容易導致知識斷裂與帶新人品質不穩。現行設計強調四週集中訓練、明確節點與三個月共筆，目的就在於把培訓變成制度，而不是全靠個人臨場帶。

    2. 第一週與後續實務必須連起來看¶

    第一週不是獨立的一段，而是後續實務的預備。若第一週工具沒建好、概念沒打底，第二週以後即使開始接案，也會一直卡在最基礎的行政與工具問題。

    3. 漸進式承接比形式平均更重要¶

    新人培訓強調的是「適合學習的入口」與「逐步授權」，而不是一開始就形式上平均分配各種案件。這是為了兼顧學習品質、個案風險與中心整體運作。

    4. 共筆與訪談不是額外附加工作¶

    共筆與小行政訪談看似增加工作量，但其實是把重複說明、重複提問與零碎知識轉成可累積的組織資產。若沒有這些機制，很多知識會持續停留在口耳相傳，無法穩定傳承。

    四、注意事項¶

    • 第一週雖不正式派案，但新人仍應主動記錄、提問與整理，不宜被動等待別人餵答案。
    • 5/11 起開始實務，不代表可以跳過前一週打底內容；若基礎未完成，應先補強而非硬撐進案。
    • 初期若遇到工具未開通、系統權限不足或案情閱讀困難，應盡快向督導或承辦反映。
    • 新人初期跟訪、共訪與電話練習的目的在於建立節奏與判斷，不應只把它當作旁聽或被動陪同。
    • 若當年度因人力、案量或報到時間不同而調整訓練順序，仍應保留「先建置、再理解、再進入實務」的基本邏輯。

    新人到職後各組小行政協助融入與主題學習¶

    一、目的¶

    本節整理新人到職後，除了督導、行政與既定課表外，各組小行政可以如何協助新人進入中心工作脈絡、認識主題線與逐步融入團隊，讓新人不只是知道要做什麼，也知道各條工作線是怎麼運作的。

    二、為什麼這篇重要¶

    新人剛到職時，最容易先看到的是： - 要交哪些資料 - 要裝哪些軟體 - 要申請哪些系統 - 要先學哪些基本規則 但如果只有這些，仍很難真的理解中心整體怎麼運作。 因為第一區的工作不是只有派案與系統登打，還包含： - 各種小行政工作線 - 家庭支持活動 - 宣導與網路社群 - 資源地圖 - 教育訓練控管 - 專業到宅 - 工作手冊與流程圖像化 - 微旅、一家心聚、社工選物誌等方案 因此，新人到職後除了督導主責帶領外，各組小行政也可成為「主題學習入口」。

    三、現行基本原則¶

    依中心現行共識內容，新人訓練以督導為主，包含： - 個案處遇 - 資源連結 - 工作整體節奏 - 基本工作分工與輪序 但同時，各組小行政也可協助新人融入主題學習，形成第二層支持。

    四、各組小行政可協助的方向¶

    1. 網路社群小行政¶

    可協助新人理解： - 中心 FB／LINE 在做什麼 - 各組素材如何收繳與發布 - LINE 名單如何辨識待處理、民眾與已分派主責 - 網路宣傳如何與方案工作連動

    2. 專業教育控管小行政¶

    可協助新人理解： - 時數怎麼算 - 研習證明怎麼命名 - 檔案放哪裡 - 每月何時填報 - 哪些課程是基本必修方向

    3. 資源地圖小行政¶

    可協助新人理解： - 中心如何整理資源 - 資源不是只有「知道名稱」，而是要分類、標註與更新 - 常見資源類別有哪些 - 如何從資源角度協助案家

    4. 工作手冊小行政¶

    可協助新人理解： - 為什麼很多流程要做成手冊 - 圖像化流程如何幫助新人快速理解 - 哪些規則已經有 SOP - 若現場做法與手冊不同時，應如何回報與修正

    5. 身障微旅小行政¶

    可協助新人理解： - 微旅為何不是單純活動，而是個案處遇資源 - 主責推薦、保險、表單、行前會、滿意度問卷怎麼串起來 - 為什麼主責推薦者就是天然工作人員

    6. 一家心聚小行政¶

    可協助新人理解： - 中心如何辦理系列宣導活動 - 實體與線上活動如何協作 - 小行政如何協助宣傳、資料整理與滿意度調查

    7. 社工選物誌小行政¶

    可協助新人理解： - 如何從案家產品故事做倡議推廣 - 紙本與網路宣傳如何並行 - 為什麼這類工作也是正式服務的一部分

    8. 團康活動小行政¶

    可協助新人理解： - 團隊凝聚活動在中心文化中的位置 - 團康與支持性工作氛圍的關聯 - 不是所有工作都只靠個人硬撐

    五、新人最適合的學習方式¶

    新人剛進來時，不適合一次塞滿所有小行政細節。較適合的方式是：

    1. 先知道「有這條線存在」¶

    先有整體概念，知道： - 這條線做什麼 - 主責是誰 - 有需要時去哪裡找資料

    2. 再依工作進度逐步進入¶

    例如： - 開始接活動前，再深入看微旅或一家心聚 - 要整理時數前，再深入看教育控管 - 開始做宣導時，再深入看網路社群與文宣流程

    3. 邊做邊學¶

    小行政最有用的地方，不只是講解，而是能讓新人在實際任務裡看到流程如何運作。

    六、小行政協助新人的基本原則¶

    各組小行政若要協助新人，應把握以下原則： - 1. 先講工作線定位 - 2. 再講最常用的路徑與表單 - 3. 再講實際最容易出錯的地方 - 4. 不要一次塞太多細節 - 5. 讓新人知道有問題可以問誰

    七、與督導主責訓練的關係¶

    小行政協助新人，不是取代督導，而是補強。 可理解成： - 督導：帶整體工作、個案處遇、資源連結與服務判斷 - 小行政：帶某條主題線的實際工具、節奏與細節 兩者合在一起，新人才比較容易真的融入中心工作。

    八、常見錯誤¶

    • 只讓新人學行政操作，沒講主題脈絡
    • 只丟資料夾路徑給新人，沒有說明用途
    • 一次把所有小行政細節全講完，反而記不住
    • 小行政各自做各自的，沒有讓新人知道整體怎麼串

    九、注意事項¶

    • 新人訓練仍以督導為主。
    • 小行政協助重點在「主題學習入口」，不是把新人立刻變成主責小行政。
    • 若某條工作線已形成固定流程，應同步導向相對應手冊文章。
    • 若未來小行政分工異動，本篇也應同步更新。
